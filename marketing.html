Produto digital
para setores exigentes.

Energia distribuída e compliance fiscal têm algo em comum: erros custam caro, dados têm de ser fiáveis e os utilizadores não têm paciência para software mal pensado. É exatamente aqui que trabalho.

Construir produto digital para energia solar ou para compliance fiscal não é como construir mais uma app. São contextos onde os dados têm de ser exatos, as integrações complexas, os utilizadores técnicos — e onde a margem para erro é mínima.

Tenho experiência real nestes dois mundos: fui responsável pelo desenvolvimento de plataformas proprietárias de monitoramento de ativos energéticos na Faro Energy, e liderei a área de Tax IT na BDO Brasil — uma das maiores firmas de auditoria do mundo. Não projeto o que não conheço.

"O melhor produto digital para o setor de energia não é o mais bonito — é o que os engenheiros de campo usam sem formação e os CFOs confiam sem reservas."

O que diferencia
produto para estes setores

O domínio do negócio vem primeiro
Em energia e fiscal, não se consegue fazer bom produto sem entender profundamente o negócio. Um engenheiro de campo que usa um dashboard tem necessidades radicalmente diferentes de um analista fiscal. Conheço os dois lados.
Dados são o produto
Nestes setores, a interface é o meio — os dados são o fim. A arquitetura de como os dados são recolhidos, processados, apresentados e auditados define a qualidade do produto. O design começa aí, não nas cores.
Fiabilidade não é opcional
Em compliance fiscal, um dado errado tem consequências legais. Em monitoramento de energia, uma falha não detetada custa dinheiro real. Construo produto para contextos onde errar tem custo — e onde a confiança é conquistada lentamente.
